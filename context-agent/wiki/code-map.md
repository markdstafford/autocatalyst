## Source tree

- `apps/control-plane/` — bootable headless Fastify control-plane service. `src/config.ts` reads `CONTROL_PLANE_PORT`/`CONTROL_PLANE_DATABASE_PATH`/`CONTROL_PLANE_BEARER_TOKEN`/`CONTROL_PLANE_MASTER_SECRET`/`AUTOCATALYST_RUN_CONCURRENCY` (default `2`) or CLI flags (`--port`, `--database-path`, `--bearer-token`, `--master-secret`, `--run-concurrency`); `src/server.ts` composes routes with persistence, unlocks `SqliteSecretStore` on startup, accepts `ControlPlaneServerOptions` (bearerToken, masterSecret, databasePath, runConcurrency, optional policy/health/extensionRegistry/providerAdapters/onProviderComposition, optional `unitOfWork: RunUnitOfWork` and `onControlPlaneReady(service)` for tests, optional `realRunnerDispatch: { enabled, defaultProviderProfileId }`), wires `InMemoryRetainedRunEventStore`, `RunDispatchQueue`, `DefaultOrchestrator`, and `DefaultControlPlaneService`; reads persisted configuration records once during startup, invokes `composeConfiguredProviders`, exposes the structured result through `onProviderComposition`, and logs sanitized production diagnostics through `logProviderCompositionDiagnostics`; when `realRunnerDispatch.enabled=true`, auto-registers the Claude adapter (`createClaudeAgentAdapter`) into the adapter map if not already present, builds a `AgentProviderAdapterRegistry` via `composeAgentProviderAdapterRegistry`; **routing-first dispatch:** builds `createModelRoutingResolver` from tenant-scoped configuration records and adapter registries, exposes `createRoutingProfileResolver(options)` which bridges `AgentRunnerFactoryInput`/`DirectCallFactoryInput` to the model routing resolver (validates role with `sessionRoleSchema`, throws `ModelRoutingConfigurationError('route_not_found')` for invalid role); production dispatch is routing-first, falls back to `defaultProviderProfileId` only when `routing_table_missing` AND an explicit profile is configured; also wires `createExplicitProfileResolver` (looks up `defaultProviderProfileId` configuration record and builds a `ResolvedAgentRunnerProfile`/`ResolvedAgentCredentialReference`), creates a `AgentRunnerFactory` via `createAgentRunnerFactory`, wraps it in a `createDelegatingExecutionEntryPoint` (adds temp-dir scratch root for `workspace.shape='none'` runs), and chains through `createExecutionContextResolver` + `createExecutionRunUnitOfWork`; `resolveRole` seam defaults every step to `'implementer'` (per-step role catalog not yet wired — see `decisions/model-routing-resolve-role-default.md`); exported helpers: `createExplicitProfileResolver`, `createRoutingProfileResolver`, `logProviderCompositionDiagnostics`, `createControlPlaneServer`, `startControlPlaneServer`; `src/main.ts` is the executable entrypoint; `src/integration.spec.ts` proves health, probe-resource persistence across restart, SSE, degraded database health, startup provider composition with fake adapters, and orchestrator ingress end-to-end over the network (`POST /v1/conversations`, `GET /v1/runs/:id`, `GET /v1/runs/:id/steps`, SSE `GET /v1/runs/:id/events` with a real `run_state_transition` frame after a tick); `src/control-plane-service.integration.spec.ts` proves the same end-to-end path at the service level with a real SQLite DB + `DrizzleConversationIngressRepository` + `DefaultOrchestrator` + `InMemoryRetainedRunEventStore` + `DefaultControlPlaneService`.
- `packages/api-contract/` — shared API contract package. `src/health.ts`, `src/probe-resource.ts`, `src/errors.ts`, and `src/sse.ts` own Zod schemas, inferred types, route constants, and status constants. `src/principal.ts` owns `principalKindSchema`, `principalSchema`, `principalDiagnosticResponseSchema`, and `principalDiagnosticPath = '/v1/principal'`. `src/secret.ts` owns `secretHandleSchema`, `secretCollectionPath = '/v1/secrets'`, `createSecretRequestSchema`, and `createSecretResponseSchema`. `src/configuration-record.ts` owns full CRUD Zod schemas: `createConfigurationRecordRequestSchema`, `configurationRecordResponseSchema`, `configurationRecordListResponseSchema`, `updateConfigurationRecordRequestSchema`, `deleteConfigurationRecordSuccessStatusCode`, and `configurationRecordCollectionPath = '/v1/configuration-records'`; `CreateConfigurationRecordRequest`, `UpdateConfigurationRecordRequest`, and `ConfigurationRecord` are now kind-tagged discriminated unions with `tenant` field; routing-specific additions: `ModelRoutingTableSettings`, `UpdateModelRoutingTableSettings` (routing table config with entries and roleDistinctRequirements), `AgentModelRouteKey`, `DirectModelRouteKey`, `ModelRouteKey` (discriminated union on route key type), `ModelRoutingEntry`, `RoleDistinctRequirement` (routing entry and distinct-role constraint), `ModelRoutingErrorCode` (typed error code enum: `route_not_found`, `routing_table_missing`, `role_distinct_unsatisfied`, `ambiguous_route`, `invalid_route_key`). `src/errors.ts` also exports error code constants: `unauthorizedErrorCode`, `validationErrorCode`, `notFoundErrorCode`, `secretStoreLockedErrorCode`. `src/openapi.ts` generates OpenAPI from those schemas and constants. Domain entity schemas (all Zod, all with inferred TS types): `src/domain-value-objects.ts` — shared value objects: `ModelIdentity`, `TokenBreakdown`, `Cost`, `TrackedIssue`, `ChannelReference`, `FeedbackAnchor`, `FeedbackThread`, `TestingGuideResult`, `InferenceSettings`, `SessionRole`, `FrontedResource`, `CredentialReference`, `JsonValue`, `NonModelPrincipal`; also exports `requireTenantMatchesOwner` helper. `src/project.ts` — `Project` + `CreateProjectInput`. `src/conversation.ts` — `Conversation` + `CreateConversationInput` (nullable `activeTopicId`). `src/topic.ts` — `Topic` + `CreateTopicInput` (`kind: main|side`, no `isActive` marker). `src/message.ts` — `Message` + `CreateMessageInput` (`direction: inbound|outbound`). `src/run.ts` — `Run` + `CreateRunInput` (`currentStep`: extensible string, `terminal`: bool; no `status`/`RunStatus`). `src/artifact.ts` — `Artifact` + `CreateArtifactInput` (`kind`, `canonicalRecord`, `cachedStatus` enums). `src/feedback.ts` — `Feedback` + `CreateFeedbackInput` (`target`, `status` enums, optional `anchor`, `thread`). `src/publication.ts` — `Publication` + `CreatePublicationInput` (`frontedResource`). `src/pull-request.ts` — `PullRequest` + `CreatePullRequestInput` (`state: open|merged|closed`). `src/run-step.ts` — `RunStep` + `CreateRunStepInput` (no owner/tenant — parent-hung). `src/session.ts` — `Session` + `CreateSessionInput` (`tokens` must equal `cost.tokens`; no owner/tenant). `src/test-result.ts` — `TestResult` + `CreateTestResultInput` (no owner/tenant). Runner event vocabulary: `src/runner-events.ts` owns `runnerEventTypeSchema`, `runnerEventImportanceSchema`, `runnerTerminalDirectiveSchema`, `runnerEventSchema` (discriminated union on `type`), and inferred types `RunnerEvent`, `RunnerAssistantTurnEvent`, `RunnerToolActivityEvent`, `RunnerProgressEvent`, `RunnerNotificationEvent`, `RunnerStepCheckpointEvent`, `RunnerTerminalResultEvent`. Post-validation handoff schema: `src/step-results.ts` owns `RunnerTerminalStepResult` and `runnerTerminalStepResultSchema` — these are the post-validation execution-to-core handoff schemas (`advance` with optional `result: Record<string,unknown>`, `needs_input`, `fail`); separate from the raw runner terminal result in `runner-events.ts`; used by `execution-boundary-events.ts` terminal event and consumed by core's `execution-run-unit-of-work.ts`. Declarative execution context: `src/execution-context.ts` owns `executionContextSchema`, `workspaceProvisioningIntentSchema`, `secretBindingSchema`, and inferred types `ExecutionContext`, `WorkspaceProvisioningIntent`, `SecretBinding`; also exports `skillRefSchema` (reference to a named skill by catalog key), `resolvedSkillSchema` (a skill with its resolved asset path), and `skillIntentSchema` (the union of skill intents used in execution context). Tests: `__tests__/domain-value-objects.spec.ts`, `__tests__/domain-entities.spec.ts`. Public entry point: `packages/api-contract/src/index.ts`.
- `packages/core/` — control-plane core package. Depends on `@autocatalyst/execution` (vite alias for tests, workspace dep in package.json). Execution-adapter modules: `src/runner-event-stream.ts` owns `consumeRunnerEventStream` which validates schema (via `runnerEventSchema.safeParse`), enforces single-terminal protocol, checks runId, wraps `onEvent` hook throws as `RunnerProtocolError('runner_failed')`, and throws `RunnerProtocolError` for all protocol violations; `src/execution-run-unit-of-work.ts` owns `createExecutionRunUnitOfWork` which wires `resolveContext → execute → validateExecutionBoundaryEventStream` (from `@autocatalyst/execution`) and maps terminal directives (advance/needs_input/fail) to `RunWorkResult`; `advance` terminal events carry an optional `result: Record<string,unknown>` field that is forwarded to `RunWorkResult.advance.result`; `RunnerProtocolError` from protocol violations are re-thrown, runner errors before terminal map to a static, sanitized `{ directive: 'fail', reason }` using the error's type/code only (never raw `error.message`, which may contain sensitive data): `ExecutionMaterializationError` → `'Execution failed: <code>'`; all others → static string; `src/execution-context-resolver.ts` owns `createExecutionContextResolver(options)` factory which builds a declarative `ExecutionContext` from `RunWorkInput` — maps work kinds to workspace shapes (feature/enhancement/bug/chore → two_roots, question → none, file_issue → scratch_only), validates secret envName format, and throws `ExecutionContextResolutionError` with codes: `unsupported_work_kind`, `unsupported_workspace_shape`, `missing_project`, `missing_workspace_settings`, `invalid_secret_declaration`, `resolver_unavailable`. `src/secret.ts` (augmented) additionally exports `SecretResolutionErrorCode`, `SecretResolutionError` class, `SecretResolver` interface, and `SecretResolutionErrorDetails`. New ingress modules: `src/orchestrator.ts` owns `Orchestrator`/`DefaultOrchestrator` with `createRun`, `createConversationWithFirstRun`, `applyDirective`, `dispatch`, `tick`, the `RunUnitOfWork` seam (`{ run(input: RunWorkInput): Promise<RunWorkResult> }`), and `OrchestratorError`/`OrchestratorErrorCode` (`active_run_conflict`/`missing_run`/`terminal_run`/`invalid_transition`/`unknown_work_kind`/`forbidden`/`persistence_failed`); `src/run-events.ts` owns `RunEventBus`/`RunEventPublisher`/`RunEventSubscriber`, `RunEventSubscription`, `createRunStateTransitionEvent`, and the live-only `InMemoryRunEventBus` (per-(runId,tenant) subscribers, no durable replay; - `packages/openai-agent-adapter/` — `@autocatalyst/openai-agent-adapter`, the `{openai, agent}` runner cell, integrated with the **real** `@openai/agents` SDK (0.11.6; deps: `@openai/agents`, `openai`, `zod`@4 — the adapter pins zod 4 for the SDK while the rest of the repo stays on zod 3.25, see `decisions/openai-agent-adapter-sdk-integration.md`). Implements `AgentProviderAdapter` with `providerKind='openai'`, `adapterId='openai-agents-sdk'`, `supportedConnectionMechanism='fetch_transport'`.
  - **Per-session binding, never global.** `startSession` builds a per-session `OpenAI` client whose `fetch` is bridged to `connection.createFetchTransport()` (`bridgeTransportToFetch`), wraps it in `new OpenAIProvider({ openAIClient, useResponses: false })`, and hands that to a per-session `new Runner({ modelProvider, tracingDisabled: true })`.   - **Sandbox + workspace materialization.** `UnixLocalSandboxClient({ workspaceBaseDir, snapshot })` is created per session with an explicit `NoopSnapshotSpec` (validated via `isNoopSnapshotSpec`; type must be `'noop'` or `sandbox_snapshot_unsupported` is thrown). Each declared workspace root becomes a `localDir({ src: root })` `Manifest` entry **plus** an `extraPathGrants` entry (the local sandbox otherwise rejects `local_dir` sources outside its base dir). `client.create(manifest)` opens the session; `runner.run(agent, prompt, { sandbox: { session } })` drives it. Workspace roots that escape the declared set fail with `workspace_containment_violation`.
  - **Event mapping.** The run driver yields the SDK's `RunItem`s; `mapRunItem` maps `message_output_item`→`runner_assistant_turn`, `tool_call_item`→`runner_tool_activity` (or progress/notification for `update_plan`/`report_progress`/`notify`), `tool_call_output_item`→`runner_tool_activity` result. Tests inject fakes for dispatch wiring; the package's own spec drives the **real** module with only the OpenAI client's `fetch` mocked. `sandboxWorkspaceBaseDir` points the default local sandbox at a temp dir.

Orchestrator service ingress routes (added by Story 7-9):

- `POST /v1/conversations` — create a conversation, topic, optional message, run, and initial run step atomically; returns the created `Conversation`/`Topic`/`Message?`/`Run`/`RunStep`. Body schema: `createConversationWithFirstRunRequestSchema` from `@autocatalyst/api-contract`.
- `GET /v1/runs/:id` — return the run for the authenticated tenant. 404 if missing; 403 if cross-tenant.
- `GET /v1/runs/:id/steps` — list run-steps for a run ordered by `(startedAt, id)`. 404/403 like above.
- `GET /v1/runs/:id/events` — Server-Sent Events stream of `ClientRunEvent` frames for the given run+tenant. Streams full runner event vocabulary (assistant turns, tool activity, progress, notifications, step checkpoints, terminal results) plus state transitions. Route subscribes before replaying so no events are lost during reconnect; replayed events are written first, then live events from the subscription. Each frame has `event: <type>`, `id: <eventId>`, and `data: <json>` lines. `Last-Event-ID` header triggers replay-after-cursor; unknown or expired cursors return HTTP `409 Conflict` with JSON `{ error: { code: "run_event_replay_cursor_unknown" | "run_event_replay_cursor_expired", message, lastEventId } }` before any SSE bytes (no `: connected` comment).

## Package generation path

```bash

PACKAGE_SCOPE=adapter
pnpm nx g @nx/js:library "packages/${PACKAGE_NAME}" \
  --bundler=tsc \
  --unitTestRunner=vitest \
  --linter=eslint \
  --importPath="@autocatalyst/${PACKAGE_NAME}" \
  --strict=true \
  --minimal=true
```
